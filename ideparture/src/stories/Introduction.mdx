# Welcome to iDeparture's Storybook

Welcome to the **iDeparture App** Storybook documentation. This platform is your guide to understanding and exploring the reusable UI components built for the iDeparture app, designed to help travelers plan their trips with ease.

## What is iDeparture?

The iDeparture app is a feature-rich utility designed to help travelers determine the optimal time to leave for the airport, considering factors like:

- Travel time to the airport
- Check-in duration
- Security screening
- Boarding time
- Pre-flight requirements

This application ensures users arrive at the airport in time for their flights and improves their travel experience by minimizing stress.

## Key Features

- **Atomic Design Principles**: The app’s components follow Atomic Design, ensuring scalability and maintainability.
- **Reusable Components**: Built with reusable components such as buttons, inputs, forms, and more.
- **Storybook Integration**: All components are showcased and tested in isolation using Storybook.
- **Dark Mode and Light Mode UI support**.

## Storybook Sections

Explore the following sections to learn more about the UI components and design principles used in iDeparture:

- **Atoms**: The basic building blocks of the UI, such as buttons and form inputs.
- **Molecules**: Simple combinations of atoms like input fields with labels.
- **Organisms**: More complex components like navigation bars and forms.
- **Templates & Pages**: Full-page designs that guide the user experience.
